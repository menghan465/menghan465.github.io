## 篇 1：时代错配 — 别用部落志给 2096 年的地图写文明

### 适合发表

✅ 工程实践 / 游戏工具 / LLM 应用

### 问题

Azgaar Fantasy Map Generator 支持年历，但常用的 D-PLACE / Ethnographic Atlas 收录的主要是小规模社会（焦点年偏 19–20 世纪）。直接用它当 prior 会让近未来国家的文化叙事退化成「游牧/渔猎/村社」。

### 做法

- 读取 `options.year`，分 5 个 era bucket（ancient → near_future）。
- 近现代 / 近未来 → Natural Earth + WorldClim + World Bank WDI 参考单元。
- 古代 → premodern 功能原型（帝国/城邦/封建/航海等合成类型）。
- D-PLACE 降为可选项，不再是默认。
- 统计截面钳定（year=2096 → WDI 截面 ~2020）。

### 局限

- WDI 是国家截面，不是区域网格；代理字段存在（如电力）。
- premodern 是**合成**功能型，不是真实 Seshat 编码。
- 宗教只有 Pew 宏标签（粗分类），不到教派级。

### 技术关键词

era routing, bucket-aware prior, D-PLACE, WDI, data proxy

## 篇 2：双通道经济 prior — 为什么 26 国不能全是服务业

### 适合发表

✅ 内容管线 / procedural narrative / 游戏工具

### 问题

WDI 中多数中高收入国家以服务业为主。如果直接用 WDI 的 `agri/industry/services` 当 prior，26 个国家几乎全是「服务业 + 办公室」叙事。但地图上有高山、深海、荒漠、洪泛平原——地理差异被统计抹平了。

### 做法

  发展侧（WDI）：
    - 锁定「时代与水平」：城市化 / GDP / 产业占比。

  地理侧（Azgaar geo_sector）：
    - 港口、河流农业、干旱畜牧、高地采掘、内陆物流、都会服务。
    - 权重由海岸比 / 海拔 / 河流 / 港口 burg 数计算。

  合成：
    - `narrative_focus = {primary, secondary}`
    - 硬规则：coast<0.08 禁航运；elev 高 + pop 低禁 CBD。

  文案：
    - 主部门 + 地理次部门双写，不是 argmax(services)。

### 效果（Bubin 例）

- 高海岸国 primary → port（如 Szazseletia）
- 高低地国 primary → extractive（如 Jungia）
- 大河都会国 primary → metro + agri（如 Pondetim）
- 不再全图 services-only。

### 局限

- geo_sector 是手工权重，不是推导的经济模型。
- 近未来 metro 仍然偏多（WDI 分布所致）。
- 不是「解决所有同质化」，是「显著减少」。

### 技术关键词

geo-channel, narrative focus, WDI, Azgaar, dual prior, generation constraint

## 篇 3：三层生成 + micro LLM — 省和镇不是模板填空

### 适合发表

✅ procedural content / game tooling / LLM pipeline

### 问题

很多地图生成器只有国家级文本，一到省市就没了。或者不加区分对所有城镇打 LLM → 成本高、质量差。

### 做法

- 三层：culture core（民族母题）→ **state**（政体 + prior + LLM）→ **province**（地理 δ）× burg（site type）。
- 省/镇不从 scratch 推理：LLM prompt 带上父 state 的 summary / narrative / core_vs_periphery。
- 全图钩子去重：`_pick_hook` 用 pattern 配额（同骨架最多 2 城）。
- LLM 失败 → 静默回退模板，并在 JSON 保留 `llm_error`。
- 每州抽样 2+2 省/镇（不全量渲染），成本可控。

### 实际量级（Bubin，DeepSeek V4 Pro）

- State：26 次 API
- Province：52 次
- Burg：52 次
- 总 ~130 次 API，并行 6，几分钟到十几分钟。

### 局限

- 省/镇依赖 state 叙事一致性，偶尔漂移。
- 省/镇 LLM 不是独立 prior 推理，是继承生成。
- 模板回退存在，micro LLM 质量需抽检。

### 技术关键词

multi-layer, procedural narrative, micro LLM, inheritance, prompt constraint
